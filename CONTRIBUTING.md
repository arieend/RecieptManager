# 🤝 Contributing to Receipt Manager

We welcome contributions to the Receipt Manager project! This guide explains how to contribute.

## 🚀 Getting Started

1.  **Fork the repository** on GitHub.
2.  **Clone your fork** locally:
    ```bash
    git clone https://github.com/your-username/receipt-manager.git
    ```
3.  **Install dependencies**:
    ```bash
    npm install
    ```
4.  **Create a new branch** for your feature or bug fix:
    ```bash
    git checkout -b feature/my-new-feature
    ```

## 🛠 Development Workflow

1.  **Run the development server**:
    ```bash
    npm run dev
    ```
2.  **Write code** and add tests for your changes.
3.  **Run tests** to ensure everything is working correctly:
    ```bash
    npm test
    ```
4.  **Lint your code** to maintain code quality:
    ```bash
    npm run lint
    ```

## 📝 Coding Guidelines

- **TypeScript**: Use TypeScript for all new code.
- **Tailwind CSS**: Use Tailwind utility classes for styling.
- **Icons**: Use `lucide-react` for all icons.
- **Animations**: Use `motion` for all animations.
- **Accessibility**: Ensure all interactive elements have `aria-label` or `role`.
- **Testing**: Add tests for all new features and bug fixes.

## 📬 Submitting a Pull Request

1.  **Commit your changes** with a clear and descriptive message:
    ```bash
    git commit -m "feat: add support for PDF scanning"
    ```
2.  **Push your changes** to your fork:
    ```bash
    git push origin feature/my-new-feature
    ```
3.  **Open a Pull Request** on the original repository.
4.  **Wait for review** and address any feedback.

## 🐞 Reporting Bugs

If you find a bug, please open an issue on GitHub with the following details:
- A clear and descriptive title.
- Steps to reproduce the bug.
- Expected and actual behavior.
- Screenshots or error logs (if applicable).
- Your environment (browser, OS, etc.).

## 💡 Feature Requests

If you have an idea for a new feature, please open an issue on GitHub with the following details:
- A clear and descriptive title.
- A detailed description of the feature and why it's useful.
- Any relevant examples or screenshots.

Thank you for contributing!
