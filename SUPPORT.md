# 🆘 Support Guide

This guide explains how to get support for the Receipt Manager project.

## 🚀 Getting Started

If you're new to the project, please start by reading the following documents:
- **README.md**: Project overview and setup instructions.
- **DEVELOPMENT.md**: Technical details for developers.
- **FIREBASE.md**: Firebase configuration and security rules.
- **TESTING.md**: Testing strategy and how to run tests.

## 📬 Contact Support

If you have any questions or need help, please contact us via:
- **GitHub Issues**: For bug reports and feature requests.
- **Email**: For general questions and support at [your-email@example.com].
- **Discord/Slack**: Join our community for real-time support (if applicable).

## 🐞 Reporting Bugs

If you find a bug, please open an issue on GitHub with the following details:
- A clear and descriptive title.
- Steps to reproduce the bug.
- Expected and actual behavior.
- Screenshots or error logs (if applicable).

## 💡 Feature Requests

If you have an idea for a new feature, please open an issue on GitHub with the following details:
- A clear and descriptive title.
- A detailed description of the feature and why it's useful.

## 🛠 Troubleshooting

Before contacting support, please check the following:
- **Environment Variables**: Ensure all required environment variables are set correctly.
- **Firebase Configuration**: Ensure your Firebase project is configured correctly.
- **Google APIs**: Ensure the Gemini API, Drive API, and Sheets API are enabled.
- **Console Logs**: Check the browser console for any error messages.
- **Tests**: Run the tests to ensure everything is working correctly.

Thank you for using the Receipt Manager project!
